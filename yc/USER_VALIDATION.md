## User Interview Plan

### Target Interviewees

**Solo Podcasters (5-10 interviews)**
- 1K-50K monthly downloads
- Actively seeking sponsorships
- Using manual tools or basic analytics

**Producers/Agencies (3-5 interviews)**
- Managing 3-10 podcasts
- Need portfolio-level insights
- Using multiple tools or spreadsheets

**Brands/Sponsors (2-3 interviews)**
- Spending $10K+/quarter on podcast ads
- Need ROI proof
- Using manual tracking or ad networks

---

## Interview Questions

### Problem Validation

1. **"What's your biggest pain point with podcast monetization?"**
   - Expected: Manual report creation, can't prove ROI, attribution is guesswork

2. **"How do you currently track campaign performance?"**
   - Expected: Spreadsheets, manual tracking, unreliable promo codes

3. **"How long does it take you to create a sponsor report?"**
   - Expected: 2+ hours

4. **"What would make you switch to a new tool?"**
   - Expected: Time savings, ROI proof, automation

---

### Solution Validation

1. **"Would you pay $29/month for automated reports and ROI tracking?"**
   - Expected: Yes, if it saves 2+ hours/week

2. **"What features are most important to you?"**
   - Expected: Automated reports, ROI calculation, attribution tracking

3. **"What would prevent you from using this?"**
   - Expected: Cost, complexity, time to set up

---

### Market Validation

1. **"How many other podcasters do you know with similar problems?"**
   - Expected: Many (validates market size)

2. **"Would you refer other podcasters to this tool?"**
   - Expected: Yes, if it works (validates virality)

## Interview Template

### Pre-Interview

**Preparation:**
- Review persona profile
- Prepare questions
- Set up recording (with permission)

**Introduction:**
- "Thanks for your time. We're building a tool to help podcasters monetize better. I'd love to understand your experience."

---

### During Interview

**Structure:**
1. **Warm-up (5 min):** Background, current situation
2. **Problem exploration (15 min):** Pain points, current solutions
3. **Solution validation (10 min):** Would you use this? What's important?
4. **Market validation (5 min):** How many others have this problem?
5. **Wrap-up (5 min):** Any questions? Can we follow up?

**Tips:**
- Listen more than talk
- Ask "why" questions
- Dig into specific examples
- Avoid leading questions

---

### Post-Interview

**Follow-up:**
- Send thank you email
- Share findings (if appropriate)
- Ask for referrals
- Keep in touch for beta testing

**Documentation:**
- Transcribe key quotes
- Document pain points
- Note feature requests
- Update validation findings
