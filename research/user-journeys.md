# User Journeys: Podcast Analytics & Sponsorship Tool

## Overview

This document maps at least 3 user journeys per persona, covering key interactions from onboarding through advanced use cases. Each journey includes touchpoints, emotions, pain points, and opportunities.

---

## 1. Solo Podcaster Journeys

### Journey 1.1: Onboarding & First Setup

**Goal:** Get set up and track first sponsor campaign

**Stages:**

1. **Discovery & Sign-Up**
   - **Touchpoint:** Learns about tool from podcast community, blog, or ad
   - **Emotion:** Hopeful, slightly skeptical ("Will this actually help?")
   - **Action:** Signs up for free trial
   - **Pain Point:** Unsure if tool is worth the cost
   - **Opportunity:** Clear value prop, social proof, free trial

2. **Account Creation**
   - **Touchpoint:** Email signup, password creation
   - **Emotion:** Excited but slightly overwhelmed
   - **Action:** Creates account, verifies email
   - **Pain Point:** Another account to manage
   - **Opportunity:** Social login (Google, Apple), clear next steps

3. **Podcast Connection**
   - **Touchpoint:** Connect RSS feed or hosting platform
   - **Emotion:** Anxious ("Will this break my podcast?")
   - **Action:** Enters RSS feed URL or connects hosting account
   - **Pain Point:** Technical confusion, fear of breaking something
   - **Opportunity:** Clear instructions, video tutorial, one-click integrations

4. **Platform Integration**
   - **Touchpoint:** Connect Apple Podcasts, Spotify, etc.
   - **Emotion:** Overwhelmed ("Do I need to connect everything?")
   - **Action:** Connects 1-2 platforms (usually Apple + Spotify)
   - **Pain Point:** Multiple platforms, OAuth complexity
   - **Opportunity:** Guided setup, explain why each platform matters

5. **First Campaign Setup**
   - **Touchpoint:** Create first sponsor campaign
   - **Emotion:** Confused ("What information do I need?")
   - **Action:** Enters sponsor name, campaign dates, promo code
   - **Pain Point:** Unclear what fields are required vs. optional
   - **Opportunity:** Template campaigns, pre-filled examples, tooltips

6. **Attribution Setup**
   - **Touchpoint:** Set up tracking (promo code, pixel, etc.)
   - **Emotion:** Intimidated ("This seems technical")
   - **Action:** Enters promo code, adds tracking pixel to website
   - **Pain Point:** Technical setup, unclear instructions
   - **Opportunity:** Step-by-step wizard, copy-paste code snippets

7. **First Report Generation**
   - **Touchpoint:** Generate report for sponsor
   - **Emotion:** Relieved ("This was easier than I thought!")
   - **Action:** Clicks "Generate Report," reviews, sends to sponsor
   - **Pain Point:** Unsure if report looks professional
   - **Opportunity:** Preview before sending, customization options

**Success Metrics:**
- Time to first report: <30 minutes
- % completing setup without support: >70%
- % generating first report within 24 hours: >60%

### Journey 1.2: Launching a Sponsor Campaign

**Goal:** Set up and launch a new sponsor campaign efficiently

**Stages:**

1. **Campaign Planning**
   - **Touchpoint:** Receives sponsor inquiry or RFP
   - **Emotion:** Excited ("New revenue opportunity!")
   - **Action:** Reviews sponsor requirements, dates, deliverables
   - **Pain Point:** Need to quickly assess if campaign is worth it
   - **Opportunity:** Campaign ROI calculator, historical performance data

2. **Campaign Creation**
   - **Touchpoint:** Creates new campaign in tool
   - **Emotion:** Focused, slightly rushed
   - **Action:** Fills out campaign form (sponsor, dates, creative assets)
   - **Pain Point:** Repetitive data entry, need to remember past campaigns
   - **Opportunity:** Campaign templates, duplicate previous campaigns, auto-fill

3. **Tracking Setup**
   - **Touchpoint:** Configures attribution tracking
   - **Emotion:** Cautious ("I need to get this right")
   - **Action:** Sets up promo code, tracking pixel, UTM parameters
   - **Pain Point:** Multiple tracking methods, technical complexity
   - **Opportunity:** Unified tracking setup, one-click configuration

4. **Creative Asset Management**
   - **Touchpoint:** Uploads sponsor creative (ad copy, links, etc.)
   - **Emotion:** Organized, methodical
   - **Action:** Uploads files, enters ad copy, sets insertion dates
   - **Pain Point:** Managing multiple assets, version control
   - **Opportunity:** Asset library, version history, scheduling

5. **Campaign Launch**
   - **Touchpoint:** Activates campaign, inserts first ad
   - **Emotion:** Nervous excitement ("Will this work?")
   - **Action:** Confirms campaign is live, inserts ad in episode
   - **Pain Point:** Uncertainty about proper insertion, timing
   - **Opportunity:** Launch checklist, best practices guide, confirmation

6. **Initial Monitoring**
   - **Touchpoint:** Checks campaign performance after 24-48 hours
   - **Emotion:** Anxious ("Is it working?")
   - **Action:** Views dashboard, checks attribution data
   - **Pain Point:** Unclear if early numbers are good or bad
   - **Opportunity:** Benchmark comparisons, "early indicators" guidance

**Success Metrics:**
- Time to campaign launch: <15 minutes
- % of campaigns launched on time: >95%
- % with tracking properly configured: >90%

### Journey 1.3: Generating Sponsor-Ready Reports

**Goal:** Create professional report to send to sponsor, prove ROI, secure renewal

**Stages:**

1. **Report Trigger**
   - **Touchpoint:** Sponsor requests update or campaign ends
   - **Emotion:** Stressed ("I need to create a report, ugh")
   - **Action:** Opens tool, navigates to reporting section
   - **Pain Point:** Dread of manual report creation
   - **Opportunity:** Automated report scheduling, one-click generation

2. **Report Configuration**
   - **Touchpoint:** Selects campaign, date range, report template
   - **Emotion:** Slightly overwhelmed ("Which template? What dates?")
   - **Action:** Chooses template, customizes date range, selects metrics
   - **Pain Point:** Unclear which metrics sponsors care about
   - **Opportunity:** Recommended templates, metric explanations, sponsor-specific templates

3. **Data Review**
   - **Touchpoint:** Reviews generated report data
   - **Emotion:** Relieved ("The data looks good!")
   - **Action:** Scans report, checks numbers, looks for issues
   - **Pain Point:** Need to verify data accuracy, explain anomalies
   - **Opportunity:** Data quality indicators, anomaly detection, explanations

4. **Customization**
   - **Touchpoint:** Customizes report (adds notes, branding, etc.)
   - **Emotion:** Professional ("I want this to look good")
   - **Action:** Adds personal notes, uploads logo, adjusts formatting
   - **Pain Point:** Limited customization options, time-consuming
   - **Opportunity:** Easy customization, branding options, saved preferences

5. **Report Generation**
   - **Touchpoint:** Generates final PDF/PDF report
   - **Emotion:** Satisfied ("This looks professional!")
   - **Action:** Clicks "Generate Report," downloads PDF
   - **Pain Point:** Long generation time, file size issues
   - **Opportunity:** Fast generation (<10 seconds), optimized file sizes

6. **Report Delivery**
   - **Touchpoint:** Sends report to sponsor via email
   - **Emotion:** Confident ("I've proven my value")
   - **Action:** Attaches PDF, writes email, sends
   - **Pain Point:** Need to write professional email, track if sponsor opened
   - **Opportunity:** Email templates, tracking, automated delivery

7. **Follow-Up & Renewal**
   - **Touchpoint:** Sponsor responds, discusses renewal
   - **Emotion:** Hopeful ("Will they renew?")
   - **Action:** Reviews sponsor feedback, negotiates renewal
   - **Pain Point:** Need data to justify rate increases
   - **Opportunity:** Renewal insights, rate justification tools, comparison data

**Success Metrics:**
- Time to report generation: <5 minutes
- % of reports sent on time: >90%
- Sponsor renewal rate: >70%
- % of creators who increase rates: >40%
